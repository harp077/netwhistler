1.Introduction

    NetWhistler is an easy to use open-source Network monitoring software that offers integrated fault and performance management functionality.
    This program auto-detects networks and presents them on graphical maps, which reflect up-to-the-minute device and host status. It includes powerful diagnostic tools including ping, trace, telnet, main network services monitoring, portscan, vncviewer, email alerts, network discovery, SNMP tools (MIB browser and Trap Console for incoming SNMP traps from any SNMP-enabled device in the network), syslog server, MRTG console and etc. NetWhistler can automatically scan a network in search of non-SNMP and SNMP-enabled devices that you want to manage in your network. Scanning parameters ( network address, network mask, SNMP community ) are fully configurable.
    Once the discovery operation has been completed, the program displays a map of all devices along with basic information about them such as address, DNS name, MAC address and description. You can find or delete a node on the map, or change node properties. The map can be stored in a local xml file.

    NetWhistler can be run using icon on your desktop or netwhistler.sh script in netwhistler directory. You can increase memory for java, adding "-Xmx[MEMORY_SIZE]m" (f.e. "-Xmx256m") after "-server" option in startup script. Before running the script, the java directory should be set to the PATH. Also fping must be installed. From this version you can use your own ping script, written in Python, Perl, Ruby, Bash or others languages.

    After startup you can view Net-View or OpenView-like interface. Double-click on globe icon opens Networks Panel where you can view directly connected networks. Here you can add or delete network, discover it, add or delete router and links. Double-click opens Nodes Panel for this network. You can also use Map Tree for navigation. The Program Interface use 5 panels: Topology View (main), Status View ( real-time table), Interface View (real-time) , Events View (real-time), Graphs View (Nodes State Summary Graph and Nodes With Longest Response Time Graph). Use Map Tree Panel or Toolbar Finder to find node on Map Toplogy Panel or others.

     

2. Main Menu Commands

    Map Menu=>Map Menu=>New [Ctrl-N] : Creates a new map manually.

    Map Menu=>Open [Ctrl-O]: Opens an existing map in xml format.

    Map Menu=>Save [Ctrl-S]: Saves an opened map in xml file. When you save a map for the first time, NetWhistler prompts you to enter its name. If you want to change the name and directory of an existing map before you save it, choose the Save As command.

    Map Menu=>Save As: Saves an opened map to a specified xml filename.

    Map Menu=>Save as Image: Saves an opened map to an image.

    Map Menu=>Exit [Alt-X]: Exits from NetWhistler

    Edit Menu => Add Node [F5]: Adding node on a map. To add the node to the map, you should specify the following parameters: IP Address, the hardware type of node, what to use for label (displayed under the device icon): IP Address, DNS-name,IP Address + DNS or SNMP System name.
    In the "Info" tab you can add node description, displayed on hovering the cursor over the node's icon. Here it is possible to choose hint image associated with the node, which is displayed on hovering the cursor over the node's icon. Select checkbox on SNMP tab and write Read and Write Communities if this node is SNMP-managable. For unmanagable nodes choose hardware-type "hub", for networks - "network-cloud". After adding a node to the map, if the node is in your network, the program will establish node MAC Address. Pushing "Ok" button will add the node to the map.

    Edit Menu => Add Box [F6]: Adding box on a map. You can add text to box, apply various colors and a shadow. To delete a box, right-click the box and select "Delete".

    Edit Menu => Operations [Ctrl-G]: Opens a Map Operations dialog. Here you can change labels type for all nodes.

    Edit Menu => Options [F9]: Opens a main Map Options dialog with tabs: Monitoring, Alerts, SNMP, Colors, Types, External Commands and etc. Here it is possible to change monitoring period, number of ping retries, reply timeout (in milliseconds), services monitoring period, SNMP community for discovering, SNMP monitoring period, map colors, image to be displayed in the map background. Here it is possible to add or to remove nodes types. You must put transparent gif (48x48) with same type name in "pics" directory. It's possible to indicate what script to use for ping and MAC determining. Also it's possible to indicate the script, which will be carried out after SNMP monitoring. See examples in "scripts" directory.

    You may add external commands (such as Ping, Ftp) to the node context menu (Use 'xterm -e telnet %IP' for example or write shell script with 'read' at the end). To execute an external command, right-click the node and select the required command. Use "%IP" as node IP Address in command arguments. NetWhistler may notify you, if any node wakes up or shuts down. There are 5 methods of notification: displaying the popup alert on the screen, sending the e-mail message(html or plain text), running external command (perl script for example), saving the event from Event Console in the log-file, displaying this event on Map Status page.

    Monitoring Menu=>Start/Stop [F8]: Use this command to start/stop monitoring process.

    Monitoring Menu=>Map Status [Ctrl-M]: Opens a Map Status Table which contain the detailed information about the map.

    Monitoring Menu=>Event Log Console [Ctrl-E]: Opens a Event Log Console which shows monitoring event log.

    Monitoring Menu=>View Menu => Show/Hide Toolbar [Ctrl-B]: Shows or hides the toolbar, which includes buttons for some of the most common commands in NetWhistler, such as Wizard, Open,etc.

    Monitoring Menu=>View Menu => Show/Hide Map Tree [F2]: Shows or hides the Map Tree Panel.

    View Menu => Full Screen [F11]: Enable FullScreen Mode. To return press F11 again.

    View Menu => Zoom Out/In [Ctrl-Z]: Zooming map out/in.

    SNMP Menu=>MRTG Console [Ctrl-R]: Opens a MRTG Console (based on JRobins MRTG). You can view MRTG traffics graphs for SNMP-managable nodes.

    SNMP Menu=>Trap Console [Ctrl-T] : Opens a SNMP Trap Console (which runs Trapd server for incoming traps).

    Tools Menu=>Syslog Console [Ctrl-L]: Opens a Syslog Console which runs Syslog server for receiving syslog messages from remote nodes.

    Help=> Help [F1]: This command will open your default Web browser with this page.

    Help=>About [Ctrl-F1]: This command presents About dialog box which displays the copyright notice and version number of your copy of NetWhistler.

3. Node Context Commands

    Status: Use this command to view detailed node status. Here you can view tabs:
  1. Summary: Node summary info - IP Address, Mac Address, Hardware type, UpTime and etc;
  2. Services: Node services status;
  3. SNMP : Node SNMP main oids (for SNMP-managable nodes);
  4. SNMP Monitoring OIDs: Use this command to view SNMP Monitoring OIDs values (for SNMP-managable nodes);
  5. Network Interfaces: Use this command to view interfaces (for SNMP-managable nodes).
    Connect To...: Use this command to add a link to another Node.
    Tools: This submenu contains some commands such as telnet, portscan, vncviewer.
    Ping: Ping selected node.
    Trace: This command opens panel with node trace results.

    Monitoring => Enable(Disable): Use this command to enable or disable node monitoring.

    Monitoring =>Network Services=>Check Now: Use this command to check node services status.

    Monitoring =>SNMP =>Get OIDs Now: Use this command to get SNMP Monitoring OIDs.

    SNMP (for SNMP-managable nodes)=>MIB Browser: Opens SNMP MIB Browser foe selected node.

    SNMP (for SNMP-managable nodes)=>Identify: Use this command to identify node type. This process use "oids.xml" file from snmp directory. You can edit this xml manually.

    Delete: Use this command to delete a node.

    Properties: Opens node properties dialog (same as "Add Node").
